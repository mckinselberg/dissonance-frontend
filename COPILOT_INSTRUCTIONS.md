# GitHub Copilot Instructions

Welcome to the dissonance-frontend project! This document provides guidance for using GitHub Copilot and Copilot Chat effectively in this codebase.

## General Guidelines
- Use Copilot to suggest code completions, refactorings, and documentation.
- When using Copilot Chat, ask clear, specific questions about the codebase or features you want to implement.
- Review all Copilot suggestions for correctness and style before committing.

## Project-Specific Tips
- This project uses React (TypeScript) with Vite for frontend development.
- TypeScript strictness is enforced; ensure all types are correct.
- Use the provided scripts in `package.json` for development, build, and linting.
- Follow the existing folder structure for components, hooks, and libraries.
- When adding new features, prefer functional components and React hooks.

## Common Tasks
- **Start development server:** `npm run dev`
- **Build for production:** `npm run build`
- **Lint code:** `npm run lint`
- **Preview production build:** `npm run preview`

## Contribution
- Write clear commit messages.
- Keep code well-documented and maintainable.
- Use Copilot to generate tests and documentation where possible.

For more help, use Copilot Chat or refer to the project README.
